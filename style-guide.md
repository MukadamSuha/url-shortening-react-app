# Style Guide for URL Shortener Project

## General Guidelines
- Follow consistent naming conventions for files and components (e.g., PascalCase for components).
- Use single quotes for strings in JavaScript and JSX.
- Maintain a clean and organized folder structure.

## Code Formatting
- Use 2 spaces for indentation.
- Keep lines no longer than 80 characters.
- Use semicolons at the end of statements.

## React Component Structure
- Each component should be in its own file.
- Use functional components and React hooks where applicable.
- PropTypes should be used for type checking of component props.

## CSS Guidelines
- Use class names that are descriptive and follow a BEM (Block Element Modifier) methodology.
- Keep styles modular by using CSS modules or styled-components where possible.
- Avoid using inline styles; prefer external stylesheets.

## API Interaction
- Use async/await for handling asynchronous operations.
- Handle errors gracefully and provide user feedback.

## Testing
- Write unit tests for components and utility functions.
- Use Jest and React Testing Library for testing React components.

## Documentation
- Keep comments clear and concise.
- Update README.md with relevant information about new features or changes.
